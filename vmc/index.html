    Operations

    vmc

    locale

    Operation HTTP request Description
    set POST https://{server}/rest/vmc/api/locale Sets the locale for the session which is used for translating responses.

    vmc orgs

    orgs

    Operation HTTP request Description
    get GET https://{server}/rest/vmc/api/orgs/{org} Get details of organization
    list GET https://{server}/rest/vmc/api/orgs Return a list of all organizations the calling user (based on credential) is authorized on.

    vmc orgs account link

    account link

    Operation HTTP request Description
    get GET https://{server}/rest/vmc/api/orgs/{org}/account-link Gets a link that can be used on a customer's account to start the linking process.

    compatible subnets

    Operation HTTP request Description
    get GET https://{server}/rest/vmc/api/orgs/{org}/account-link/compatible-subnets Gets a customer's compatible subnets for account linking
    post POST https://{server}/rest/vmc/api/orgs/{org}/account-link/compatible-subnets Sets which subnet to use to link accounts and finishes the linking process

    compatible subnets async

    Operation HTTP request Description
    get GET https://{server}/rest/vmc/api/orgs/{org}/account-link/compatible-subnets-async Gets a customer's compatible subnets for account linking via a task. The information is returned as a member of the task (found in task.params['subnet_list_result'] when you are notified it is complete), and it's documented under ref /definitions/AwsCompatibleSubnets
    post POST https://{server}/rest/vmc/api/orgs/{org}/account-link/compatible-subnets-async Sets which subnet to use to link accounts and finishes the linking process via a task

    connected accounts

    Operation HTTP request Description
    delete DELETE https://{server}/rest/vmc/api/orgs/{org}/account-link/connected-accounts/{linked_account_path_id} Delete a particular connected (linked) account.
    get GET https://{server}/rest/vmc/api/orgs/{org}/account-link/connected-accounts Get a list of connected accounts

    map customer zones

    Operation HTTP request Description
    post POST https://{server}/rest/vmc/api/orgs/{org}/account-link/map-customer-zones Creates a task to re-map customer's datacenters across zones.

    sddc connections

    Operation HTTP request Description
    get GET https://{server}/rest/vmc/api/orgs/{org}/account-link/sddc-connections Get a list of SDDC connections currently setup for the customer's organization.

    providers

    Operation HTTP request Description
    list GET https://{server}/rest/vmc/api/orgs/{org}/providers

    sddc templates

    Operation HTTP request Description
    delete DELETE https://{server}/rest/vmc/api/orgs/{org}/sddc-templates/{template_id} Delete SDDC template identified by given id.
    get GET https://{server}/rest/vmc/api/orgs/{org}/sddc-templates/{template_id} Get configuration template by given template id.
    list GET https://{server}/rest/vmc/api/orgs/{org}/sddc-templates List all available SDDC configuration templates in an organization

    vmc orgs sddcs

    sddcs

    Operation HTTP request Description
    create POST https://{server}/rest/vmc/api/orgs/{org}/sddcs Provision an SDDC in target cloud
    delete DELETE https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc} Delete SDDC
    get GET https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc} Get SDDC
    list GET https://{server}/rest/vmc/api/orgs/{org}/sddcs List all the SDDCs of an organization
    patch PATCH https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc} Patch SDDC

    vmc orgs sddcs addons

    credentials

    Operation HTTP request Description
    create POST https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc_id}/addons/{addon_type}/credentials Associated a new add on credentials with the SDDC such as HCX
    get GET https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc_id}/addons/{addon_type}/credentials/{name} Get credential details by name
    list GET https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc_id}/addons/{addon_type}/credentials List all the credentials assoicated with an addon type with in a SDDC
    update PUT https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc_id}/addons/{addon_type}/credentials/{name} Update credential details

    clusters

    Operation HTTP request Description
    create POST https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc}/clusters Creates a new cluster in customers sddcs with passed clusterConfig.
    delete DELETE https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc}/clusters/{cluster} This is a force operation which will delete the cluster even if there can be a data loss. Before calling this operation, all the VMs should be powered off.

    convert

    Operation HTTP request Description
    create POST https://{server}/rest/vmc/api/orgs/{org}/sddcs/{sddc}/convert This API converts a one host SDDC to a four node DEFAULT SDDC. It takes care of configuring and upgrading the vCenter configurations on the SDDC for high availability and data redundancy.
